# 🚨 ИСПРАВЛЕНИЕ ПРЯМО СЕЙЧАС

## Шаг 1: Исправление RLS политик (2 минуты)

1. В Supabase SQL Editor откройте новый запрос
2. Скопируйте **ВЕСЬ** файл `supabase/quick-fix-all.sql`
3. Вставьте в редактор
4. Нажмите **Run** (или F5)
5. Дождитесь выполнения (должно появиться "✅ ИСПРАВЛЕНИЕ ЗАВЕРШЕНО")

## Шаг 2: Добавление тестовых данных (1 минута)

Если в результатах шага 1 вы видите "❌ Нужны данные":

1. Откройте новый запрос в SQL Editor
2. Скопируйте **ВЕСЬ** файл `supabase/test-data.sql`
3. Вставьте в редактор
4. Нажмите **Run**
5. Должно появиться:
   ```
   Мастеров добавлено: 1
   Услуг добавлено: 5
   Связей мастер-услуга: 5
   Записей в графике: 7
   ```

## Шаг 3: Добавление себя как администратора (30 секунд)

1. Узнайте свой Telegram ID:
   - Напишите боту [@userinfobot](https://t.me/userinfobot)
   - Скопируйте число (например: 123456789)

2. В SQL Editor выполните (замените на свой ID):
   ```sql
   INSERT INTO admins (telegram_id, name)
   VALUES (123456789, 'Ваше имя');
   ```

## Шаг 4: Проверка (30 секунд)

В SQL Editor выполните:
```sql
SELECT 
  (SELECT COUNT(*) FROM masters) as masters,
  (SELECT COUNT(*) FROM services) as services,
  (SELECT COUNT(*) FROM master_services) as connections,
  (SELECT COUNT(*) FROM master_schedules) as schedules,
  (SELECT COUNT(*) FROM admins) as admins;
```

Должно быть:
- masters: 1
- services: 5
- connections: 5
- schedules: 7
- admins: 1

## Шаг 6: Тестирование (1 минута)

1. Откройте бота в Telegram
2. Отправьте `/start`
3. Нажмите "Записаться на услугу"
4. Должны загрузиться 5 услуг
5. Выберите услугу → выберите мастера → выберите дату и время
6. Нажмите "Записаться"
7. Откройте консоль браузера (F12) и следите за логами

## Если не работает:

### Проблема: Услуги не загружаются

**Решение:**
1. Откройте консоль браузера (F12)
2. Посмотрите ошибки
3. Если "RLS policy violation" - выполните снова `quick-fix-all.sql`
4. Если "No rows returned" - выполните `test-data.sql`

### Проблема: Кнопка "Записаться" не работает

**Решение:**
1. Откройте консоль браузера (F12)
2. Должны быть логи "=== ОТПРАВКА ДАННЫХ ==="
3. Если их нет - проблема в коде
4. Если есть, но нет ответа от бота - проверьте что бот запущен

### Проблема: Бот не получает данные

**Решение:**
1. Проверьте что бот запущен (в консоли должно быть "✅ Бот успешно запущен!")
2. Проверьте что Mini App открыт через кнопку в боте, а не напрямую
3. Проверьте логи бота - должно быть "=== ПОЛУЧЕНЫ ДАННЫЕ ИЗ WEB APP ==="

## Быстрая диагностика:

Выполните в SQL Editor:
```sql
-- Проверка RLS
SELECT tablename, 
       CASE WHEN rowsecurity THEN '🔒 Включен' ELSE '🔓 Выключен' END as rls
FROM pg_tables
WHERE schemaname = 'public' AND tablename = 'masters';

-- Проверка политик
SELECT COUNT(*) as policies_count FROM pg_policies WHERE schemaname = 'public';

-- Проверка данных
SELECT 
  (SELECT COUNT(*) FROM masters) as masters,
  (SELECT COUNT(*) FROM services) as services;
```

Должно быть:
- RLS: 🔒 Включен
- policies_count: больше 10
- masters: 1
- services: 5
