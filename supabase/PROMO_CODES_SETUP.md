# Настройка промокода QWERTY11

## Шаги для применения изменений

### 1. Примените миграцию для многоразовых промокодов

Откройте SQL Editor в Supabase Dashboard и выполните содержимое файла:
```
supabase/migrations/013_reusable_promo_codes.sql
```

Эта миграция добавляет поддержку многоразовых промокодов:
- Делает `client_telegram_id` необязательным
- Добавляет поля `is_reusable`, `usage_limit`, `usage_count`

### 2. Добавьте промокод QWERTY11

После успешного применения миграции, выполните:
```
supabase/add-promo-qwerty11.sql
```

Это создаст промокод:
- Код: `QWERTY11`
- Скидка: 11%
- Срок действия: 1 год
- Многоразовый: да
- Лимит использований: без ограничений

### 3. Проверка

После выполнения скриптов проверьте результат:
```sql
SELECT * FROM promo_codes WHERE code = 'QWERTY11';
```

## Типы промокодов

Теперь система поддерживает два типа промокодов:

### Одноразовые (персональные)
- Привязаны к конкретному клиенту (`client_telegram_id`)
- Используются один раз (`is_used = true` после использования)
- Пример: промокоды "возвращения" для неактивных клиентов

### Многоразовые (общие)
- Не привязаны к клиенту (`client_telegram_id = NULL`)
- Можно использовать много раз (`is_reusable = true`)
- Опциональный лимит использований (`usage_limit`)
- Счётчик использований (`usage_count`)
- Пример: QWERTY11

## Как работает валидация

Когда клиент вводит промокод, система:
1. Сначала ищет многоразовый промокод с таким кодом
2. Проверяет срок действия и лимит использований
3. Если не найден многоразовый, ищет одноразовый для этого клиента
4. При использовании многоразового промокода увеличивается счётчик
5. При использовании одноразового он помечается как использованный
