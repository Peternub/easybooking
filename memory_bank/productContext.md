# Product Context

## Для чего нужен проект

EasyBooking нужен для того, чтобы салон мог принимать записи клиентов через привычный для аудитории Telegram-интерфейс без отдельного сайта и при этом сохранять операционный контроль у администратора.

## Проблемы, которые решает продукт

- ручная запись в мессенджерах и звонках плохо масштабируется;
- администратору сложно быстро видеть загрузку мастеров и историю клиентов;
- без автоматических напоминаний растет вероятность пропусков визитов;
- сбор отзывов после визита без автоматизации происходит нерегулярно.

## Основные пользователи

- клиент салона, который хочет быстро выбрать услугу, мастера и свободное время;
- администратор салона, который управляет расписанием, каталогом услуг, клиентской базой и отзывами;
- владелец или менеджер, которому нужна прозрачная картина загрузки и статусов записей.

## Ключевая ценность MVP

- быстрый self-service сценарий записи в Mini App;
- единое хранилище записей и клиентов в PostgreSQL;
- автоматические Telegram-коммуникации вокруг визита;
- административный контур для ежедневной работы салона.
